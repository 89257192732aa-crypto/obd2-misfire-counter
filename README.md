# OBD-II Misfire Counter & Car Diagnostic Tools (Python)

Набор скриптов на Python для взаимодействия с электронным блоком управления (ECU) автомобиля через диагностический интерфейс OBD-II (адаптеры ELM327 Bluetooth/Wi-Fi). Основная цель проекта — логирование параметров двигателя и подсчет пропусков зажигания (Misfires).

> **Примечание по разработке:** Логика утилит спроектирована мной самостоятельно для решения реальных повседневных задач. Скрипты разработаны в парадигме **AI-assisted development** (с использованием ИИ для ускорения написания кода). Проекты адаптированы, протестированы и поддерживаются мной вручную.

## 🛠️ Структура проекта и архитектура скриптов:

* **`obd_core.py`** — Базовое ядро проекта. Содержит логику подключения к адаптеру, отправку стандартных PID-запросов и расшифровку HEX-ответов от автомобиля.
* **`obd_simulator.py`** — Имитатор (эмулятор) электронного блока автомобиля. Позволяет тестировать интерфейс программы на ПК без подключения к реальной машине (генерирует фальшивые потоки данных: обороты, пропуски, температуру).
* **`obd_test_pc.py`** — Скрипт для тестирования связи и отладки команд через ноутбук при прямом подключении к машине.
* **`obd_test_android.py` / `obd_mobile_app.py`** — Экспериментальные модули для переноса логики на мобильные устройства (смартфоны на Android) с целью использования приложения в качестве бортового мини-компьютера.

## 🧰 Технологический стек:
* Python 3
* Протоколы OBD-II / CAN-bus (работа с PID кодами)
* Библиотеки для работы с Bluetooth/Serial (например, `pyserial` или встроенные обертки)
